# Accessibility Checklist

## 화면과 텍스트
- 모든 주요 화면의 사용자 문구는 한국어를 기본으로 유지한다.
- 버튼 텍스트는 동작을 직접 설명한다. 예: `주행 시작`, `문의 접수`, `전체 읽음 처리`.
- 오류 문구는 원인과 다음 행동을 함께 제공한다.
- 동적 텍스트는 작은 화면에서 줄바꿈이 가능해야 하며 버튼 안에서 잘리지 않아야 한다.

## 터치와 포커스
- 주요 CTA는 최소 48px 이상의 터치 영역을 유지한다.
- 주행 중 화면은 버튼 수를 최소화하고 광고, 팝업, 도전장, 알림을 표시하지 않는다. 종료 확인도 모달 대신 같은 화면 안의 2단계 버튼 상태로 제공한다.
- 권한 거부 화면은 앱 설정 이동 경로를 제공한다.

## 색상과 상태
- 점수, 검증, 보상, 위험 상태는 색상만으로 구분하지 않고 텍스트 배지를 함께 제공한다.
- 공정성/개인정보 안내는 정확한 GPS 좌표가 공개되지 않음을 명확히 설명한다.

## 테스트 기준
- onboarding, login, consent, setup, drive start, result, support, settings 화면 렌더링을 mock provider로 확인한다.
- TalkBack/VoiceOver 수동 점검에서 하단 탭, 주요 버튼, 권한 화면, 문의 입력 필드 순서를 확인한다.
