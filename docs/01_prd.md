# PRD

## 요구사항
사용자는 한국어 앱에서 가입, 로그인, 필수 동의, 권한 안내, 차량 등록, 주행 기록, 결과 확인, 랭킹, 배틀, 시즌 미션, 광고 보상, 프리미엄, 쿠폰, 설정 관리를 수행할 수 있어야 한다.

## 사용자 시나리오
- 신규 사용자는 온보딩 후 가입하고 필수 동의와 차량 등록을 마친 뒤 홈에 진입한다.
- 로그인 사용자는 대표 차량과 시즌 상태를 확인하고 바로 주행을 시작한다.
- 주행 후 사용자는 점수 분석, 검증 상태, 랭킹 변화, 보상 선택을 확인한다.
- 배틀 사용자는 비금전 보상 조건으로 경쟁하고 결과 후 복수전을 신청할 수 있다.

## 기능 요구사항
- Riverpod 기반 mock/Supabase repository 분리
- go_router 기반 전체 route 구성
- foreground 주행 기록과 안전 모드 UI
- 광고 보상과 mock purchase 흐름
- 관리자 `/admin` 대시보드

## 비기능 요구사항
- service_role key 클라이언트 금지
- drive_points 비공개
- dev mode mock fallback
- production mode 설정 오류 표시
- Flutter Web 빌드 가능 구조 유지

## 성공 지표
DAU, MAU, 검증 주행 수, 랭킹 참여율, 배틀 생성 수, 광고 완료율, 프리미엄 전환율, 쿠폰 사용률, 신고 처리율.
