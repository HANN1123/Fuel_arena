# PRD

## 요구사항
사용자는 한국어 앱에서 Google 로그인, 필수 동의, 추가 설정, 차량 선택, 주행 기록, 결과 확인, 랭킹, 배틀, 시즌 미션, 광고 보상, 프리미엄, 쿠폰, 설정 관리를 수행할 수 있어야 한다.

## 사용자 시나리오
- 신규 사용자는 온보딩 후 Google로 로그인하고 필수 동의와 추가 설정 안내를 거쳐 홈에 진입한다.
- 차량 설정은 로그인 직후 강제하지 않으며, 홈/랭킹/배틀/시즌/설정에서 필요할 때 `/setup/vehicle`로 이동한다.
- 로그인 사용자는 대표 차량과 연료 리그, 시즌 상태를 확인하고 바로 주행을 시작한다.
- 주행 후 사용자는 점수 분석, 검증 상태, 랭킹 변화, 보상 선택을 확인한다.
- 배틀 사용자는 비금전 보상 조건으로 경쟁하고 결과 후 복수전을 신청할 수 있다.

## 기능 요구사항
- Riverpod 기반 mock/Supabase repository 분리
- go_router 기반 전체 route 구성
- 이메일/비밀번호 UI 없이 Google OAuth 단일 로그인 제공
- 제조사 → 모델/파생모델 → 기준 연식 → 엔진·미션 파워트레인 → 확인 순서의 차량 설정 스텝퍼
- 가솔린/디젤/하이브리드/전기차/LPG/플러그인 하이브리드/기타 리그 분리
- foreground 주행 기록과 안전 모드 UI
- 광고 보상과 프리미엄 구매 검증 fallback 흐름
- 관리자 `/admin` 대시보드

## 비기능 요구사항
- service_role key 클라이언트 금지
- drive_points 비공개
- dev mode mock fallback
- production mode에서 Supabase와 Google OAuth 설정 오류 표시
- Flutter Web 빌드 가능 구조 유지

## 성공 지표
DAU, MAU, 검증 주행 수, 랭킹 참여율, 배틀 생성 수, 광고 완료율, 프리미엄 전환율, 쿠폰 사용률, 신고 처리율.
