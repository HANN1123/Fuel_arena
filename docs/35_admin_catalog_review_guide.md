# 관리자 차량 카탈로그 검수 가이드 (docs/35)

본 문서는 Fuel Arena 관리자 앱의 각 전용 검수 화면을 활용하여 차량 데이터를 검증, 승인, 반려 및 출처 충돌 해결하는 운영 지침을 제공합니다.

---

## 1. 사용자 직접 입력 차량 검수 (AdminCustomVehicleReviewScreen)

사용자가 카탈로그에 본인 차량이 없어 직접 수동 기재하여 차량 등록을 신청한 경우, 해당 데이터는 즉시 `custom_vehicle_requests` 테이블에 `pending` 상태로 인서트되며 공식 리그 집계에서 격리됩니다.

* **검수 프로세스**:
  1. 관리자는 **Custom Vehicle Review** 화면으로 이동합니다.
  2. 신청된 수동 제원(제조사, 모델, 연식, 변종명, 연료 등)을 검토합니다.
  3. **Fuzzy matching 후보** 목록에서 기존 공식 카탈로그에 존재하는 유사 파워트레인이 있는지 비교합니다.
  4. 매칭되는 공식 파워트레인이 존재한다면 해당 `powertrain_id`를 연결하고 **[승인]** 처리합니다. 승인 시 사용자의 차량은 즉시 공식 등급으로 승격되어 리그 참여 권한을 획득합니다.
  5. 일치하는 파워트레인이 없으나 정상 제원인 경우, 관리자는 이를 신규 공식 카탈로그로 추가 등록하여 승인하고, 악성 기재 또는 거짓 제원의 경우 **[반려]** 처리하여 반려 사유(예: "배기량 기재 오기")를 기입합니다.

---

## 2. 출처 충돌 처리 (AdminPowertrainConflictScreen)

KEA 연비 데이터 일괄 임포트나 제조사 스펙 업데이트 시, 기존 DB 값과 새로 인입된 값 간의 제원 불일치가 감지되면 `vehicle_catalog_conflicts`에 수록됩니다.

* **해결 수단**:
  - **기존 값 유지**: 이미 유저들에게 서비스되어 안정적인 기존 제원을 고수합니다.
  - **신규 값 덮어쓰기**: 새로 수집된 출처가 더 정확하거나 최신 버전일 경우 신규 사양으로 강제 치환합니다.
  - **수동 병합**: 두 값의 장점을 취하여 수동으로 값을 수정한 후 충돌 상태를 `resolved`로 전환합니다.
* **이력 보존**:
  - 관리자가 승인, 반려, 수정하는 모든 액션은 자동으로 `vehicle_catalog_change_logs` 테이블에 타임스탬프, 관리자 ID, 상세 변경 전후 데이터와 사유가 로깅됩니다.
