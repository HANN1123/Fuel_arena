# Behavior Spec

## 로그인
로그인은 Google 계정만 지원한다. dev mode에서 Google OAuth 값이 없으면 mock 로그인으로 흐름을 검증하고, production mode에서 Web/Android/iOS/Server Google OAuth client ID, iOS reversed client ID, `fuelarena://login-callback` callback 설정이 누락되거나 형식이 맞지 않으면 설정 오류 화면을 표시한다.

## 차량 설정
차량은 제조사 → 모델/파생모델 → 기준 연식 → 엔진·미션 파워트레인 → 확인 순서로 설정한다. 기준 연식은 클릭 후 펼쳐지는 피커/스크롤 리스트에서 선택하며, 해당 연식의 엔진·미션 파워트레인만 다음 단계에 노출한다. 대표 차량이 설정되면 fuel_league와 vehicle_class를 계산해 profile, user_vehicles, league_memberships에 반영한다. 판매 트림과 휠 인치수는 공식 리그 분류 축으로 쓰지 않는다. 카탈로그에 없는 차량은 검수 대기 상태로 접수하고 공개 랭킹에 바로 반영하지 않는다.

## 주행 기록
주행 시작 시 대표 차량과 오늘의 미션을 확인한다. 대표 차량이 없으면 차량 설정 CTA를 보여준다. 안전 모드에 들어가면 주행 시간과 거리만 표시하고 광고와 알림을 차단한다. 종료 확인은 팝업 대신 화면 안의 `한 번 더 눌러 종료` 단계로 처리한다.

최근 주행 기록은 세션 요약과 점수 검증 상태를 목록으로 표시한다. 분석 상세는 거리, 시간, 평균 효율, 점수 패널티/보너스, 검토 요청 CTA를 제공하되 정확한 좌표와 상세 경로는 표시하지 않는다.

## 점수 계산
효율 점수, 안정 점수, 거리 보너스, 일관성 보너스에서 급가속, 급제동, 공회전 패널티를 제외한다. 비현실적 기록은 pending_review 상태가 된다.

## 랭킹 반영
verified score만 rankings에 반영한다. 공개 화면에는 닉네임, 티어, 점수, 차급, 연료 리그만 표시한다. 정확한 좌표와 raw drive_points는 공개하지 않는다.

랭킹은 가솔린, 디젤, 하이브리드, 전기차, LPG, 플러그인 하이브리드, 기타 리그를 분리한다. `내 리그`는 대표 차량의 fuel_league와 vehicle_class 기준으로 계산한다.

`/ranking/detail` 랭킹 상세 화면은 `내 리그` 랭킹 데이터를 사용해 상위 랭커, 내 현재 순위, 주변 순위를 보여준다. 공개 항목은 닉네임, 티어, 점수, 차급, 연료 리그로 제한한다.

라이벌 화면은 공개 랭킹 row만 사용해 내 위치와 바로 위 추월 목표를 보여준다. 점수 차이는 공개 점수 기준으로만 계산하고, 상세 주행 경로나 개인 식별 정보는 표시하지 않는다.

`/profile/:userId` 공개 프로필은 `public_rankings` row에서 가져온 닉네임, 티어, 점수, 차급, 연료 리그만 표시한다. `profiles` 전체 row를 직접 읽지 않고, 이메일, 정확한 위치, 상세 주행 경로, raw drive_points는 표시하지 않는다. 공개 랭킹 row가 없으면 랭킹으로 복구하는 빈 상태를 보여준다.

## 검토 요청
점수 보류, 랭킹 미반영, GPS 품질, 배틀 정산 이의는 검토 요청 화면에서 접수한다. 사용자는 문의 상세에서 답변을 확인하고, 운영자는 같은 요청을 Support Tickets와 Reports 섹션에서 함께 처리한다.

## 고객지원 FAQ
`/support/faq`는 정적 안내 화면으로 끝나지 않고 문제 유형별 복구 행동을 제공한다. 점수/랭킹 이슈는 `/support/review-request`로 이동해 검토 요청을 접수하고, 광고 보상/차량/쿠폰 이슈는 `/support/contact?category=` 쿼리로 문의 접수 화면에 초기 카테고리를 전달한다.

## 빈 상태 복구
사용자 화면은 비어 있는 데이터만 보여주지 않고 다음 행동으로 복구한다. 통계가 없으면 첫 주행 시작으로, 쿠폰이 없으면 리워드 광고로, 스폰서 챌린지가 없으면 주행 시작으로, 프리미엄 요금제가 없으면 고객지원 문의로 연결한다. 로딩과 오류 상태는 빈 상태와 분리해 표시하고 retry가 가능한 화면은 provider invalidate로 다시 시도한다.

## 배틀 정산
배틀은 현금성 보상을 사용하지 않는다. 시즌 XP, 배지 조각, 쿠폰 응모권 같은 비금전 보상만 지급한다. 공개 매칭은 같은 연료 리그와 차급을 우선하고, 다른 리그와 겨루는 경우 친선전으로 표시한다.

`/battle/detail/:battleId`와 `/battle/result/:battleId`는 목록의 첫 항목으로 대체하지 않고 URL의 `battleId`로 단일 배틀을 조회한다. 배틀이 없거나 접근할 수 없으면 배틀 목록 CTA가 있는 빈 상태를 보여준다. dev/mock mode에서도 새로 생성한 배틀은 mock 저장소에 반영되어 생성 직후 상세 화면으로 이동할 수 있어야 한다.

배틀 결과가 아직 정산되지 않았으면 결과 화면에서 `settle_battle` Edge Function으로 정산 요청을 보낸다. 요청은 idempotency key로 중복 정산을 방지하고, 성공 후 배틀 상세/목록 provider를 갱신해 완료 상태와 보상 CTA를 표시한다.

## 광고 보상
사용자가 선택할 때만 리워드 광고를 실행한다. 광고를 보지 않아도 기본 보상은 유지된다.

## 쿠폰
리워드 지갑은 만료 전 쿠폰을 보여주고 사용자가 선택한 쿠폰만 발급한다. 발급 요청은 `CouponRepository.issueCoupon`을 통해 `issue_coupon` Edge Function으로 전달하며, 같은 쿠폰 중복 발급은 서버의 `user_coupons` unique index와 idempotency key로 보호한다. 운영 설정에서 쿠폰 기능이 꺼져 있으면 발급 버튼 대신 안내 상태를 표시한다.

## 프리미엄 접근
프리미엄은 dev 모드 구매 검증 fallback 또는 in_app_purchase 상품 ID로 활성화한다. 광고 제거, 고급 통계, 라이벌 분석, 시즌패스 추가 보상이 열린다.

## 미션 진행
주행, 배틀, 광고 보상, 스폰서 챌린지 참여가 mission_progress를 증가시킨다.
