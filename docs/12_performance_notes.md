# Performance Notes

## 앱 시작
- Splash에서 로컬 상태, 보안 저장소, 현재 사용자만 복구한다.
- 홈 진입 후 네트워크 상태, 원격 설정, 주요 데이터 provider가 각자 비동기로 로드된다.
- dev/mock mode에서는 외부 키 없이도 동일한 사용자 흐름을 통과해야 한다.
- 사용자 앱 화면은 어떤 데스크톱 폭에서도 430px 이하 모바일 레이아웃으로 제한해 모바일 기준 렌더링 비용과 레이아웃 변동을 줄인다.

## 주행 기록
- 주행 중에는 광고, 팝업, 도전장, 알림을 표시하지 않는다.
- 오프라인이면 주행 세션 요약과 업로드 실패한 drive_points batch를 로컬 큐에 임시 저장하고 온라인 복귀 후 private table로 업로드한다.
- raw drive_points는 본인 상세 검증에만 사용하고 공개 랭킹/프로필에는 노출하지 않는다.

## 랭킹과 배틀
- 공개 랭킹은 제한 view를 사용해 닉네임, 티어, 점수, 차급, 연료 리그만 내려받는다.
- 연료 리그와 차급 필터를 서버 쿼리 조건으로 유지한다.
- 배틀은 공식전과 친선전을 분리해 다른 리그 비교가 공식 랭킹에 영향을 주지 않게 한다.

## 캐시와 재시도
- drive_points 재시도 큐는 성공한 batch만 uploaded로 집계한다. 손상된 payload나 지원하지 않는 legacy item은 `discarded` 로그로 남긴 뒤 제거하며, 큐 크기는 제한해 장기 미업로드 시 저장 공간을 보호한다.
- `offline_queue` 저장 JSON 자체가 손상되면 원본을 `offline_queue_corrupt_backup`에 격리하고 빈 큐로 복구한다. 일부 row만 손상된 경우 정상 item은 유지해 사용자 주행 데이터 손실을 줄인다.
- 로그아웃 시 오프라인 queue, 세션 ID 매핑, 손상 queue 백업을 지워 사용자별 로컬 주행 데이터가 다른 세션으로 이어지지 않게 한다.
- 원격 설정은 실패 시 기본값을 사용하고 화면 진입을 막지 않는다.
- 분석 이벤트는 실패해도 사용자 흐름을 중단하지 않는다.
- 지원 티켓은 Supabase 실패 시 dev/mock fallback에서 흐름 검증을 계속할 수 있다.

## 관리자 화면
- 관리자 대시보드만 full width를 사용하며, 운영 섹션은 좌측 sidebar와 가로 scroll data table로 데스크톱 반복 작업에 맞춘다.
- 관리자 목록은 검색, 상태 필터, chart summary, action menu를 같은 컴포넌트로 사용해 화면별 rebuild 범위를 줄인다.
- production 연동 시 운영 table은 서버 페이지네이션과 기간 필터를 우선 적용하고, raw drive_points나 정확한 좌표는 관리자 목록에도 기본 노출하지 않는다.
